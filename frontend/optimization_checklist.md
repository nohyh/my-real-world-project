# 现代 React 项目优化清单 (Modern React Best Practices)

这份清单旨在帮你把项目打磨成一个**标准的、高质量的现代 React 应用**。我们不追求极致的性能跑分，而是专注于**代码的可读性、可维护性**以及**业界通用的最佳实践**。

## 1. 状态管理与性能 (State & Performance)
*修复明显的性能缺陷，遵循 React 的基本规则。*

- [ ] **AuthContext 修复** (必须做)
    - **问题**: `value` 对象没缓存，导致全站组件无意义重渲染。
    - **方案**: 使用 `useMemo` 包裹 `Context.Provider` 的 `value`。

- [ ] **路由懒加载 (Lazy Loading)**
    - **方案**: 使用 `React.lazy` 引入页面。这是现代 SPA 的标配，虽然项目小，但习惯要养好。

## 3. 代码规范与工程化 (Code Quality)
*让代码看起来像是一个团队协作的产物，而不是个人 Demo。*

- [ ] **常量管理**
    - **方案**: 杜绝魔法字符串/数字。建立 `src/constants.js`，管理 `API_URL`, `PAGE_LIMIT` 等。

- [ ] **工具函数分离**
    - **方案**: 将 `formatTags`, `formatDate` 等纯函数移入 `src/utils/`。不要在组件里写复杂的辅助逻辑。

- [ ] **绝对路径导入 (Optional)**
    - **方案**: 配置 Vite alias (`@/components/...`)，避免 `../../../../` 这种深层嵌套路径，让引用更清晰。

## 4. 用户体验细节 (UX Polish)
*细节决定成败，这些是现代应用该有的样子。*

- [ ] **优雅的加载状态**
    - **方案**: 使用简单的骨架屏 (Skeleton) 或 Loading Spinner，替换纯文字 "Loading..."。

- [ ] **错误处理**
    - **方案**: 当 API 失败时，给用户一个友好的 Toast 提示或者错误版块，而不是让页面崩坏或没反应。

- [ ] **空状态处理 (Empty States)**
    - **方案**: 当列表为空时（如"没有文章"），展示一个友好的插画或提示文案。

---
**总结**: 完成以上修改后，你的代码将具备极高的可读性和扩展性，完全符合企业级 React 项目的代码标准。
